# Security Policy

## Supported Versions

Security fixes are provided for the latest release.

| Version | Supported |
| --- | --- |
| 1.x | Yes |

## Reporting a Vulnerability

Use GitHub's private vulnerability reporting feature when the repository is
published. Do not open a public issue for vulnerabilities involving command
execution, package integrity, privilege escalation, or release artifacts.

Reports should identify the affected version, reproduction steps, expected
impact, and any proposed mitigation.
